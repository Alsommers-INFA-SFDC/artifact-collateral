⚠️ Eligibility Controls (Regulation Relationships)

Eligibility controls are implemented as relationships between Regulation Categories and Locations/Banners. These relationships govern which products can be promoted at specific stores.

📍 Regulation → Location
Regulation: STATE_LAW_INDIANA_IC_7.1-5-10-25
Applies To: All locations in Indiana
Restriction: alcohol_promotion_eligible = FALSE
Example Impact:
❌ Kroger #456 (Indianapolis): No alcohol coupons
❌ Dollar General #8765 (Fort Wayne): No alcohol coupons
🏢 Regulation → Banner → Location
Regulation: PROP_65_TOBACCO
Applies To: All California locations
Banner Filter: All banners operating in CA
Restriction: tobacco_advertising_allowed = FALSE
Example Impact:
❌ Walmart #1234 (Los Angeles, CA): No tobacco ads
❌ All Kroger stores in CA: No tobacco promotions
🏷️ Banner Policy → Location
Banner: Harris Teeter (Premium Tier)
Policy: BANNER_POLICY_ORGANIC_ONLY
Restriction: Only organic/premium product placements
Effective: All Harris Teeter stores
Example Impact:
✅ Harris Teeter #2345: Premium organic chips only
❌ Budget/value chip brands: Not eligible for placement
🗺️ Category → Region → Location
Category: ALCOHOL
Region: Kroger Midwest Region
Restriction: No Sunday alcohol sales before noon
Applies To: IN, OH stores
Example Impact:
⏰ Kroger #456 (IN): Alcohol coupons blocked Sun 6am-12pm
✅ Kroger #789 (TX): No Sunday restrictions
🔗 Relationship Implementation

Eligibility controls are stored in a many-to-many relationship table:

eligibility_control_relationship
├── control_code (FK → eligibility_control_reference)
├── applies_to_location_id (FK → location) [OPTIONAL]
├── applies_to_banner_id (FK → banner) [OPTIONAL]
├── applies_to_geography_id (FK → geography_reference) [OPTIONAL]
├── effective_from_date
└── effective_to_date

Cascading Logic: Geography-level controls (state laws) automatically apply to all locations within that geography. Banner-level controls apply to all stores under that banner. Location-specific overrides can be added for exceptions.

🌳 Hierarchies

  • Organizational Hierarchy (Banner → Store)
    Organization (Parent Retailer) → Banner → Location (Store)
    Example: The Kroger Co. → Harris Teeter → Store #2345 (Charlotte, NC)
    Levels: 3 levels
    Temporal Support: Yes (tracks M&A, e.g., Harris Teeter acquired by Kroger in 2014)
    Relationship Table: org_location_relationship
  • Geographic Hierarchy (Corporate Standard)
    Country → State → Store
    Example: United States → Indiana → Kroger Store #456
    Levels: 3 levels
    Purpose: State-level regulatory controls (alcohol, tobacco laws)
    Relationship Table: geo_location_relationship (hierarchy_owner_id = NULL)
  • Parallel Geographic Hierarchy (Retailer-Specific Regions)
    Organization → Retailer Region → Store
    Example: The Kroger Co. → Kroger Midwest Region → Store #456
    Levels: 3 levels
    Key Feature: Same store exists in multiple regional groupings (Kroger Midwest vs. DG Central)
    Relationship Table: geo_location_relationship (hierarchy_owner_id = ORG_KROGER)

🔗 Relationships

  • Banner → OPERATES → Store
    Banner (1) → (N) Location
    Cardinality: 1:N (One banner operates many stores)
    Example: Harris Teeter (banner) → Store #2345, Store #2346, Store #2347
    Temporal: Yes (effective_from_date, effective_to_date)
    Attributes: relationship_type, hierarchy_level, change_reason
  • Location → LOCATED_IN → Geography
    Location (N) → (1) Geography
    Cardinality: N:1 (Many stores in one state/region)
    Example: Kroger #456, Kroger #789, DG #8765 → Indiana (State)
    Parallel Support: hierarchy_owner_id enables retailer-specific regional views
    Attributes: geo_hierarchy_type, hierarchy_owner_id, effective dates
  • Regulation → APPLIES_TO → Location
    Eligibility Control (N) → (M) Location
    Cardinality: N:M (Many-to-many, one regulation affects many stores)
    Example: STATE_LAW_INDIANA_IC_7.1-5-10-25 → All Indiana stores
    Cascading: Geography-level controls auto-apply to child locations
    Attributes: control_code, applies_to_location_id, effective dates
  • Regulation → APPLIES_TO → Banner
    Eligibility Control (N) → (M) Banner
    Cardinality: N:M (Many-to-many, banner-specific policies)
    Example: BANNER_POLICY_ORGANIC_ONLY → Harris Teeter (all stores)
    Inheritance: Banner-level controls apply to all stores under that banner
    Attributes: control_code, applies_to_banner_id, effective dates
  • Regulation → APPLIES_TO → Geography
    Eligibility Control (N) → (M) Geography
    Cardinality: N:M (Many-to-many, state/regional laws)
    Example: PROP_65_TOBACCO → California (all counties and stores)
    Cascading: Regulations cascade from geography → all locations within
    Attributes: control_code, applies_to_geography_id, effective dates
