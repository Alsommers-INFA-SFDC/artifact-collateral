1️⃣ Super SKU (SSKU) / Super Duper Items

Non-sellable ghost parent items that group multiple sellable variants. The parent exists only for organizational purposes and cannot be purchased directly.

Real-World Example: A pillow product available in multiple colors (blue, yellow, red) and sizes (small, medium, large). The "Pillow" Super SKU itself is non-sellable, but each color-size combination is a sellable variant.

Key Characteristics:

  • 🔴 Parent: Non-Sellable (Ghost/Virtual Item)
  • 🟢 Children: Sellable Variants
  • Groups by attributes: color, size, material, style
  • Simplifies product browsing and merchandising

2️⃣ Combo - Aggregated Cost with Both Parent and Children Sellable

Parent/child grouping where the parent cost is automatically aggregated from component costs and quantities. Both the combo package and individual components can be sold separately.

Real-World Example: A tool kit containing a drill ($50), drill bits set ($15), and carrying case ($20). The kit can be sold as a combo for $85, or customers can buy each component individually.

Key Characteristics:

  • 🟢 Parent: Sellable
  • 🟢 Children: Sellable
  • 💰 Cost Aggregation: Parent Cost = Σ(child costs × quantities)
  • Automatic recalculation when component costs change
  • Flexible quantity multipliers per component

3️⃣ Multibox - Single Product Shipped in Multiple Boxes

A single sellable product that ships across multiple non-sellable boxes, each with its own cost allocation. Used for oversized items that cannot fit in a single shipping container.

Real-World Example: A large sectional sofa ($1,299) that ships in 4 separate boxes: Box 1 (sofa base - $400), Box 2 (cushions - $300), Box 3 (back panels - $400), Box 4 (legs & hardware - $199).

Key Characteristics:

  • 🟢 Parent: Sellable (the complete product)
  • 🔴 Children: Non-Sellable (shipping boxes)
  • Each box has individual cost and weight
  • Logistics tracking per box
  • Total product cost = sum of all box costs

4️⃣ VAS (Value-Added Service) - Custom-Cut or Custom-Mixed Products

Non-sellable parent representing a base product where child cost copies to the parent base cost. Used for products that require customization or value-added services like cutting, mixing, or custom sizing.

Real-World Example: Window blinds that come in a default size but can be cut to custom dimensions. The default 72"×48" blind size is non-sellable, but customers can purchase custom cuts like 36"×48" ($45), 24"×36" ($30), or 60"×72" ($85).

Key Characteristics:

  • 🔴 Parent: Non-Sellable (base/default size)
  • 🟢 Children: Sellable (custom cuts/sizes)
  • 💰 Child cost copies to parent base cost
  • Common for blinds, paint, lumber, fabric
  • Supports custom dimension entry

5️⃣ Prepack - Non-Sellable Display Contains Sellable Items

Non-sellable cardboard display or merchandising fixture that contains sellable component items. This is a store concept only, used for impulse purchases, seasonal displays, or checkout merchandising.

Real-World Example: A checkout display containing candy bars, gum, and batteries. The display itself (cardboard fixture) is not sold, but customers can pick individual items from it. Common at checkout lanes for impulse purchases.

Key Characteristics:

  • 🔴 Parent: Non-Sellable (display fixture)
  • 🟢 Children: Sellable (individual items)
  • Store concept only (not for e-commerce)
  • Merchandising and planogram planning
  • Inventory tracked at item level, not display level

6️⃣ Packaging Hierarchy - Each/Box/Case/Pallet Supply Chain

Each/Case/Pallet GTIN-UPC supply chain relationships with nested quantity aggregation. Only the "Each" unit is typically sellable to consumers; higher levels are logistics and wholesale units.

Real-World Example: A bottle of shampoo sold individually (1 Each), packaged in boxes (6 Each per box), shipped in cases (4 boxes = 24 Each per case), and palletized (50 cases = 1,200 Each per pallet). Only the individual bottle is sold to consumers; cases and pallets are for distribution.

Key Characteristics:

  • 🟢 Each: Sellable (consumer unit)
  • 🔴 Box/Case/Pallet: Non-Sellable (logistics units)
  • 📦 Nested quantity aggregation
  • Each level has unique GTIN/UPC
  • Supply chain and warehouse management

Relationship Type Details

1️⃣ Super SKU (SSKU)

Non-sellable ghost parent item that groups multiple sellable variants by attributes like color, size, or material.

Parent: Non-Sellable Children: Sellable

2️⃣ Combo

Parent/child grouping where parent cost is aggregated from component costs and quantities. Both parent and children are sellable units.

Example: Meal combo with burger ($10), fries ($4), and drink ($3) = $17 total.

Parent: Sellable Children: Sellable Cost Aggregation

3️⃣ Multibox

A single product shipped across multiple boxes, each a non-sellable child with its own cost allocation.

Example: Large furniture item shipped in 3 separate boxes.

Parent: Sellable Children: Non-Sellable

4️⃣ VAS (Value-Added Service)

Non-sellable parent (base product) where child cost copies to the parent base cost. Used for customizable items.

Example: Blinds or paint that can be cut/mixed into custom sizes or colors.

Parent: Non-Sellable Children: Sellable Cost Copy

5️⃣ Prepack

Non-sellable cardboard display that contains sellable component items. Store concept only for merchandising.

Example: Floor display containing multiple products for impulse purchase.

Parent: Non-Sellable Children: Sellable

Technical Implementation Details

Cost Aggregation Patterns

  1. Combo: Parent cost = SUM(child costs × quantities). Real-time calculation based on component changes.
  2. VAS: Child cost copies to parent base cost. One-way sync on child cost update.
  3. Multibox: Each box has own cost, parent shows total. Independent box cost management.
  4. Packaging Hierarchy: Cost typically at Each level only. Higher levels are logistics containers.

Quantity Aggregation Patterns

  1. Packaging Hierarchy: Nested multiplication (1 Each → 10 Box → 100 Case → 1000 Pallet). Automatic calculation through hierarchy.
  2. Combo: Quantity multipliers affect cost calculation. Example: 2× burger + 1× fries + 1× drink.

Sellability Rules

  • Sellable Parent, Sellable Child: Combo only (both can be sold independently)
  • Non-Sellable Parent, Sellable Child: Super SKU, VAS, Prepack (parent is organizational only)
  • Sellable Parent, Non-Sellable Child: Multibox, Packaging Hierarchy (parent is the only purchasable unit)
  • External Reference: Bundle (components may exist outside IDM)
